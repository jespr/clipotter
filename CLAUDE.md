# ClipOtter

A **macOS** app (despite the repo path saying `iOS/`) for transcribing video/audio
on-device with Apple's Speech framework, then reworking the transcript with an LLM
(Apple on-device or Claude API). SwiftUI + AppKit. Ships outside the App Store with
Sparkle auto-updates. Not sandboxed.

## Project layout & build model

- **The `.xcodeproj` is generated by [XcodeGen] and gitignored** — never hand-edit
  `ClipOtter.xcodeproj/project.pbxproj`; edits get blown away. The source of truth
  is `project.yml`. Source files under `ClipOtter/` are **auto-discovered**, so a
  new `.swift` file needs no project wiring — just `xcodegen generate` (the release
  script runs this for you).
- Version lives in `project.yml` → `MARKETING_VERSION` (and `CURRENT_PROJECT_VERSION`).
- Swift sources: `ClipOtter/`. Website: `website/`. Release tooling: `scripts/`.

Build / run locally:
```sh
xcodegen generate
xcodebuild -project ClipOtter.xcodeproj -scheme ClipOtter -configuration Debug build
```

## Releasing

Full details in [`RELEASING.md`](RELEASING.md). The short version:

1. **Commit your source changes first.** `scripts/release.sh` only commits the
   regenerated `website/appcast.xml` + `changelog.html` — it does **not** commit app
   code. A clean tree before releasing keeps the tag pointing at real source.
2. Add a dated section to `CHANGELOG.md` (the script parses it for the appcast and
   GitHub release notes — format matters):
   ```
   ## [0.7.0] - 2026-06-10
   - What changed
   ```
3. Bump `MARKETING_VERSION` in `project.yml` to match.
4. Run `./scripts/release.sh 0.7.0` (or `--dry-run 0.7.0` to build/sign/verify only).

The script: `xcodegen generate` → archive → export (Developer ID, hardened runtime)
→ verify signing → build DMG → **notarize + staple** → Gatekeeper-assess → tag &
push → write `website/appcast.xml` + `changelog.html`, commit, push `main` →
`gh release create` with the DMG.

Notes:
- Prereqs: `create-dmg`, `gh` (authed), `xcodegen`, a populated `.env` (gitignored),
  and the `AC_PASSWORD` notarytool keychain profile. The script preflights these.
- A "DMG staple — Error 65 / Record not found" warning right after notarization is the
  expected Apple-CDN propagation delay; the **app inside the DMG is stapled** and
  Gatekeeper still passes. Not a failure.
- Pushing to `main` is the deploy mechanism for both the appcast (so existing users
  get the Sparkle update) and the landing page — GitHub Actions (`pages.yml`)
  publishes `website/` to `https://clipotter.app/` (custom domain via
  `website/CNAME`), which is the `SUFeedURL` baked into the app.

## Conventions

- Versioning has bumped the patch number even for feature-sized releases; pick the
  bump deliberately and confirm with the user when unsure.
- Website is a single static `website/index.html` (Tailwind via CDN). Changes here
  reach users by pushing to `main`.
